# 前端视觉评审与迭代方案

> 基于 Anthropic frontend-design skill 对当前 MimoCode 设计的评审与修改方向。
> 日期：2026-06-12

---

## 1. 项目定位

| 要素 | 内容 |
|------|------|
| 项目 | 大五人格测评工具（open-personality） |
| 受众 | 普通用户，无心理学背景，在探索自我 |
| 页面核心任务 | 引导用户诚实地完成 120 道自评题目，然后呈现有洞察力的性格画像 |
| 当前配色方案 | MimoCode 多巴胺高饱和配色（5 维度色 + 渐变 + 全圆角） |

---

## 2. 当前设计评审

### 2.1 已避开的陷阱

MimoCode 方案成功避开了 AI 生成设计的三大默认外观：
- ✅ 暖白背景 + 衬线字体 + 陶土色点缀
- ✅ 深色背景 + 单一亮色
- ✅ 报纸排版 + 零圆角

### 2.2 存在的问题

依据 frontend-design skill 的指导原则，当前设计**亮点过多**：

> *"把大胆用在一个地方，让 signature 元素成为唯一让人记住的东西。"
> "编排的时刻比散落的效果更有力。"*

| 元素 | 页面 | 问题 |
|------|------|------|
| 模糊背景 blob（3 个色块浮动） | 首页 / 全部页面 | 与"人格测评"主题无关，增加认知噪音 |
| 5 色跳动点 pulse 动画 | 首页 | 每个点独立 pulse，分散注意力 |
| 渐变彩虹进度条（含 shimmer 动画） | 答题页 | 每道题都在动，120 题会造成视觉疲劳 |
| 所有按钮 hover scale 弹跳 | 全局 | 每处交互都弹，削弱了动画的冲击力 |
| 渐变标题 + 彩虹顶部边框 + 分数条 shimmer | 报告页 | 效果堆叠，层次模糊 |

### 2.3 根因

设计没有区分 **高潮时刻** 和 **常规时刻**。所有页面都用了同等级别的视觉强度，用户无法感知"哪里是重点"。

---

## 3. 修改方向

### 核心策略：把 boldness 集中到报告页

答题完成后的**报告页**是整个体验的 reward moment——用户做了 120 道题，看到的应该是一张有冲击力、值得保存和分享的结果卡。这是 signature 的归属。

首页和答题页都保持安静、克制的设计，让报告页的爆发形成反差。

### 3.1 首页 — 清除干扰

```
改动前：
  ┌─────────────────────────────────────────────┐
  │  🌐 blob-1 (粉)  │  标题(渐变)  │  🌐 blob-2 (紫)│
  │   ● ● ● ● ● (跳动色点)                    │
  │  [标准模式]  [高级模式]                    │
  │        [开始测评]                          │
  │  ──────────  OR  ──────────               │
  │  分享码输入框                              │
  └─────────────────────────────────────────────┘

改动后：
  ┌─────────────────────────────────────────────┐
  │                                             │
  │        Open Personality                     │
  │         大五人格测评工具                     │
  │                                             │
  │    [标准模式]        [高级模式]             │
  │    120题 · 10分钟     300题 · 25分钟        │
  │                                             │
  │           [ 开始测评 ]                      │
  │                                             │
  │        ── 或查看已有报告 ──                 │
  │            [输入分享码]                     │
  └─────────────────────────────────────────────┘
```

| 操作 | 说明 |
|------|------|
| 删除 3 个模糊 blob 背景装饰 | 无实际意义，占用视觉带宽 |
| 删除 5 个跳动色点 | 让"第一次见到维度色"的体验留给报告页 |
| 标题取消渐变文字 | 使用纯色（深色 `#1A1A2E`），让副标题和模式卡安静呈现 |
| 模式卡图标保留，但去掉 hover 缩放 | icon 帮助理解即可，不需弹跳效果 |

### 3.2 答题页 — 降低视觉疲劳

```
改动前：👁️ 每道题：彩虹进度条(shimmer) + 数字水印 + 选项缩放

改动后：🧘 每道题：单色进度条 + 当前维度色提示
```

| 操作 | 说明 |
|------|------|
| 彩虹渐变进度条 → 当前维度色单色进度条 | 减少色彩动态变化。题目属于哪个维度（O/C/E/A/N），进度条就用对应颜色 |
| 去掉 progress bar 的 shimmer 动画 | 答题过程需要稳定感，不需要每帧闪烁 |
| 去掉数字水印（右下角半透明大号数字） | 题号已经在进度条下方显示了 |
| 选项 hover 动画缩小强度 | hover 仅变色（不变形），选中时放大保留——因为选中是用户的主动确认 |
| 去掉 "answered-hint" 绿色勾 | 自动跳题后用户不看旧答案，勾号无意义 |

### 3.3 总览页 — 清晰、可扫描

当前总览页设计基本合理，微调：

| 操作 | 说明 |
|------|------|
| 每行只显示题号 + 已选值 | 去掉题目文本预览（120 题会撑爆屏幕），留更多空间给快速扫描 |
| 未答题显示红色圆点，不用文字 | 视觉上更快识别遗漏 |
| 提交按钮只在全部答完才完全显示 | 未答完时显示进度条提示 |

### 3.4 报告页 — signature 爆发点 ⭐

这里是整个应用**唯一应该高调的地方**。保留并强化现有设计：

| 保留 | 增强 |
|------|------|
| ✅ 彩虹顶部条纹 | → 加宽至 8px，带轻微渐入动画 |
| ✅ 渐变 MBTI 代码 | → 增加放大效果（page load 时从 0.8x → 1x 弹入） |
| ✅ 5 色维度进度条 | → shimmer 动画只在这里保留 |
| ✅ 彩色解读卡片 | → 维度色边线加粗至 6px |
| ✅ 彩蛋 banner | → 放大 icon，增加 bounce-in 动画 |

> 报告页是用户完成 120 题后的 reward——这里是唯一允许"过度设计"的地方。

## 4. 设计 Token 汇总

角色名称可以被记住。就像是五位色彩背后藏着各自的故事，它们不应只待在色盘里，而会成为用户理解自我的线索。用户在报告中首次「撞见」这些颜色时，它们不只是装饰，而是性格维度的化身。

### 4.1 色彩系统（保留 MimoCode 色值，重新分配使用范围）

| Token | 色值 | 在哪些页面出现 |
|-------|------|--------------|
| `--color-openness` | `#7B2FF7` 亮紫 | 仅报告页 + 答题页 O 维度题目提示 |
| `--color-conscientiousness` | `#00B4D8` 电光蓝 | 仅报告页 + 答题页 C 维度题目提示 |
| `--color-extraversion` | `#FFD60A` 阳光黄 | 仅报告页 + 答题页 E 维度题目提示 |
| `--color-agreeableness` | `#56CFE1` 青柠绿 | 仅报告页 + 答题页 A 维度题目提示 |
| `--color-neuroticism` | `#FF006E` 热力粉 | 仅报告页 + 答题页 N 维度题目提示 |
| `--color-bg` | `#FAFAFA` 近白 | 全部页面 |
| `--color-text` | `#1A1A2E` 深蓝黑 | 全部页面 |
| `--color-surface` | `#FFFFFF` 白 | 全部页面（卡片） |
| `--gradient-primary` | 热力粉→亮紫 | 仅用于报告页按钮和标题 |
| `--gradient-rainbow` | 5 色渐变 | 仅用于报告页（结果卡的顶部条纹） |

> **核心原则**：5 维度色和渐变是 report 场景的特权。首页和答题页看不到它们——这样当用户在结果页第一次见到彩色的雷达图和维度卡片时，会产生惊喜感。

### 4.2 字体

| 角色 | 字体 | 使用范围 |
|------|------|---------|
| Display | Playfair Display | 首页标题、报告页标题 |
| Body | Inter | 全部正文、题目 |
| Mono | JetBrains Mono | 分数、题号、MBTI 代码 |

### 4.3 动画使用规范

| 动画类型 | 使用位置 | 频次 |
|---------|---------|------|
| `bounceIn` | 页面首次加载、结果卡出现 | 每页面一次 |
| `fadeInUp` | 卡片内容依次出现 | 顺序 stagger，每页最多 3 个元素 |
| `shimmer` | 仅报告页的分数进度条 | 持续循环 |
| `hover scale` | 仅选项按钮（答题页） | 用户悬停时 |
| `pulse` | **禁用** | 替换为静态指示 |
| 模糊背景浮动 | **禁用** | 已删除 |

### 4.4 圆角系统

| Token | 值 | 使用 |
|-------|-----|------|
| `--radius-sm` | 12px | 小卡片、按钮 |
| `--radius-md` | 20px | 主卡片 |
| `--radius-lg` | 28px | 模态窗、大区域 |
| `--radius-full` | 9999px | 语言切换开关、圆形按钮 |
