# open-personality — 决策摘要

> 阶段二~五的不可变更的决策边界。
> 如后续实现与此矛盾，直接引用本条纠偏，不复议。

---

| 编号 | 决策内容 | 来源 |
|------|---------|------|
| B-01 | 项目是 Web 测评工具，不做 App/CLI | 用户确认 |
| B-02 | 人格模型用大五人格（Big Five），不做其他模型 | 用户确认 |
| B-03 | 问卷用 IPIP-120（标准），IPIP-300（高级模式可选） | 用户确认 |
| B-04 | 问卷题目是 Public Domain，可自由使用/翻译/修改 | 法律事实 |
| B-05 | MBTI 推断基于 McCrae & Costa (1989) 概率化映射，输出置信度而非硬分类 | 用户确认 |
| B-06 | 技术栈：后端 Python FastAPI，前端 Vue 3 + Vite + ECharts | 用户确认 |
| B-07 | 数据库起步用 SQLite，按需升级 PostgreSQL | 用户确认 |
| B-08 | 微信小程序未来用 uni-app 迁移 | 用户确认 |
| B-09 | 双语言架构，右上角一键切换 zh/en，中文版做趣味性微调 | 用户确认 |
| B-10 | 不需要用户注册登录 | 用户确认 |
| B-11 | UI 极简+卡片风格，结果卡片可导出图片 | 用户确认 |
| B-12 | 彩蛋概率 10%，形式为短文案 | 用户确认 |
| B-13 | 彩蛋由用户参与编写（10~15 条短文案 + 3~5 条中等） | 用户确认 |
| B-14 | 部署方案待阶段四再决定 | 用户确认 |
| B-15 | 跨设备查历史用分享链接（share_token），不做账号登录；同设备通过 localStorage 最近记录找回 | 阶段二确认 |
| B-16 | 后端用 SQLAlchemy 2.0 ORM，换数据库时不用改业务代码 | 阶段二确认 |
| B-17 | 分享 token 用 8 位 base62 随机字符串，碰撞概率极低 | 阶段二确认 |
| B-18 | 大五分数标准化用 T-score（平均值 50，标准差 10） | 阶段二确认 |
| B-19 | 前端路由用 Vue Router hash 模式，分享链接直接可访问 | 阶段二确认 |
| B-20 | 数据库 SQLite 起步，ER 结构为 sessions→answers / sessions→reports | 阶段二确认 |
| B-21 | 所有问卷题目、MBTI 权重、解读文本、彩蛋都存为 JSON 配置文件，不改代码即可修改内容 | 阶段二确认 |
| B-22 | 前后端各持一份语言包：后端驱动解读文字，前端驱动 UI 界面 | 阶段二确认 |
